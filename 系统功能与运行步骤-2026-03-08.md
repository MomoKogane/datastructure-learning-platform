## 1. 本版结论

本版本在既有教学功能稳定运行的基础上，补齐了**数据库迁移能力**，可实现以下一键流程：

- 导出当前 MongoDB 全量数据与索引为 `mongosh` 可执行脚本
- 传输迁移脚本到目标环境（本地复制或 `scp`）
- 在目标库重建集合与索引并回灌数据
- 按集合进行源/目标文档数校验

本版重点新增：

- `back/src/utils/exportMongoSnapshotScript.ts`
- `back/scripts/mongo-migrate-oneclick.ps1`
- `back/scripts/mongo-migrate-oneclick.md`
- `back/package.json` 新命令：
  - `export:mongo:snapshot`
  - `migrate:mongo:oneclick`

## 2. 本版新增能力（数据库迁移）

### 2.1 Mongo 快照导出脚本

文件：`back/src/utils/exportMongoSnapshotScript.ts`

能力：

- 连接源库并扫描非系统集合
- 导出集合数据（保留 BSON 类型，基于 EJSON）
- 导出索引定义并写入重建逻辑
- 输出 `mongosh` 可执行文件（可视作 Mongo 版“SQL 初始化脚本”）

命令：

```powershell
cd back
npm run export:mongo:snapshot
```

常用参数：

```powershell
npx ts-node src/utils/exportMongoSnapshotScript.ts --db=dslp --batchSize=1000 --dropTarget=true
```

### 2.2 一键迁移总脚本（PowerShell）

文件：`back/scripts/mongo-migrate-oneclick.ps1`

流程覆盖：

1. 导出
2. 传输
3. 目标重建
4. 文档数校验

支持模式：

- `local`：目标库可直接访问，脚本本机恢复
- `scp`：先传到远端，再远端执行恢复与校验

命令入口：

```powershell
cd back
npm run migrate:mongo:oneclick -- -SourceMongoUri "mongodb://localhost:27017/dslp" -TargetMongoUri "mongodb://目标主机:27017/dslp"
```

## 6. 运行前准备

1. 安装 Node.js（建议 18+）
2. 安装 MongoDB 并确保服务可访问
3. 安装依赖

```powershell
npm install
cd front; npm install
cd ..\back; npm install
```


4. 配置 `back/.env`（至少）

建议：
- 敏感信息（如 AI_API_KEY、SMTP_USER、SMTP_PASS）不要上传到公开仓库。
- 上传时用占位符（如 YOUR_API_KEY、your_email@example.com、your_email_password）替换真实值，或只上传 .env.example 文件。

主要变量示例：
```env
MONGODB_URI=mongodb://localhost:27017/dslp
PORT=3001
HOST=0.0.0.0
FRONTEND_PORT=5178
FRONTEND_URL=http://localhost:5178
CONTENT_DB_NAME=dslp
AI_API_KEY=YOUR_API_KEY
SMTP_HOST=smtp.example.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your_email@example.com
SMTP_PASS=your_email_password
SMTP_FROM="Your Name <your_email@example.com>"
```

迁移脚本可额外使用：
- `TARGET_MONGODB_URI`（目标库 URI）

## 7.1 局域网访问修正

如果希望系统启动后，同一局域网下的其他设备也能访问完整功能，需要同时满足以下条件：

1. 前端必须监听局域网地址

本项目 `front/vite.config.ts` 已设置：

```ts
server: {
  port: 5178,
  host: true
}
```

2. 后端必须监听非本地回环地址

在 `back/.env` 中设置：

```env
HOST=环境变量设置（如 0.0.0.0）
PORT=环境变量设置（如 3001）
FRONTEND_PORT=环境变量设置（如 5178）
FRONTEND_URL=环境变量设置（如 http://localhost:5178）
```

3. 前端不能把接口地址写死为 `localhost`

现已改为按当前浏览器访问主机自动解析 API 地址：

- 在自己电脑打开 `http://localhost:5178` 时，请求会走 `http://localhost:3001/api`
- 在局域网设备打开 `http://192.168.x.x:5178` 时，请求会自动走 `http://192.168.x.x:3001/api`

如需手工指定后端地址，可在前端环境变量中配置：

```env
VITE_API_BASE_URL=http://192.168.x.x:3001/api
```

或：

```env
VITE_API_PORT=3001
```

4. 访问时必须使用宿主机的局域网 IP，而不是访问设备自己的 `localhost`

例如宿主机 IP 为 `192.168.1.20`，则其他设备应访问：

```text
前端：http://192.168.1.20:5178
后端健康检查：http://192.168.1.20:3001/api/health
```

5. Windows 防火墙需要放行 5178 和 3001 端口

否则即使服务已监听，局域网设备也无法连通。

6. 启动方式

```powershell
npm run dev
```

启动后，用宿主机局域网 IP 进行联调即可。

## 8. 数据库迁移操作手册（新增）

### 8.1 仅导出迁移脚本

```powershell
cd back
npm run export:mongo:snapshot
```

导出产物：`back/exports/mongo-snapshot-<db>-<timestamp>.mongosh.js`

### 8.2 本地直连一键迁移（推荐）

```powershell
cd back
npm run migrate:mongo:oneclick -- `
  -SourceMongoUri "mongodb://localhost:27017/dslp" `
  -TargetMongoUri "mongodb://10.0.0.8:27017/dslp" `
  -DbName "dslp" `
  -TransferMode local `
  -DropTarget $true
```

### 8.3 远端主机一键迁移（scp + ssh）

```powershell
cd back
npm run migrate:mongo:oneclick -- `
  -SourceMongoUri "mongodb://localhost:27017/dslp" `
  -TargetMongoUri "mongodb://localhost:27017/dslp" `
  -DbName "dslp" `
  -TransferMode scp `
  -SshTarget "user@target-host" `
  -TransferTargetPath "/tmp" `
  -DropTarget $true
```

### 8.4 跳步参数（排障常用）

- `-SkipExport`
- `-SkipTransfer`
- `-SkipRestore`
- `-SkipVerify`

说明：可用于断点续跑，例如“已导出，仅重放 + 校验”。

### 8.5 校验规则

- 脚本会拉取源库与目标库所有非系统集合的 `countDocuments({})`
- 逐集合比较文档数
- 任意集合不一致即失败并报错

### 8.6 回滚建议

- 推荐先对目标库做独立备份
- 若本次迁移异常，优先回放目标库备份
- 或重新执行一次 `-DropTarget $true` 的完整迁移流程
